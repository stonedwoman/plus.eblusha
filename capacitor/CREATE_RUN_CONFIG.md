# Создание конфигурации запуска (пошагово)

## Шаг 1: Добавить новую конфигурацию

В диалоге "Run/Debug Configurations":
1. Нажмите на **"Add new..."** (синяя ссылка) или на кнопку **"+"** (плюс) вверху слева
2. Появится меню с типами конфигураций

## Шаг 2: Выбрать тип конфигурации

В появившемся меню выберите:
**Android App** (НЕ Java, НЕ Application!)

## Шаг 3: Заполнить параметры

После выбора "Android App" откроется форма. Заполните:

### Обязательные поля:

1. **Name:** `app`
   - Просто название конфигурации

2. **Module:** `android.app`
   - Выберите из выпадающего списка
   - Должно быть: `android.app` или `capacitor.android.app`

3. **Launch:** `Default Activity`
   - Выберите из выпадающего списка
   - Это запустит MainActivity автоматически

4. **Target:** 
   - `USB Device` - если подключен телефон
   - `Emulator` - если используете эмулятор
   - Или выберите конкретное устройство из списка

### Опциональные поля (можно оставить по умолчанию):

- **Launch Flags:** оставить пустым
- **Before launch:** можно добавить "Gradle-aware Make" если нужно

## Шаг 4: Сохранить

1. Нажмите **OK** внизу диалога
2. Конфигурация появится в списке слева

## Шаг 5: Запустить

1. Закройте диалог (если он еще открыт)
2. Вверху Android Studio, рядом с кнопкой Run (▶️)
3. В выпадающем списке выберите **app**
4. Нажмите **Run** (▶️)

## Визуальная подсказка:

```
Run/Debug Configurations
├─ [Search поле]
├─ [+] [-] [Sort]  ← Нажмите [+]
│
└─ В меню выберите: Android App
   ├─ Name: app
   ├─ Module: android.app
   ├─ Launch: Default Activity
   └─ Target: [ваше устройство]
```

## Если Android App не видно в списке:

1. Убедитесь, что проект синхронизирован: `File → Sync Project with Gradle Files`
2. Проверьте, что модуль `android.app` существует в проекте
3. Попробуйте закрыть и заново открыть диалог конфигураций
